The "Failed to Fetch" error is almost certainly a Port or Protocol mismatch between your Vite frontend and the spawned API server. Here is the breakdown of why your save button is failing and how to fix it:

1. The "Hidden" Port Problem
In your electron/main.ts, you are spawning an API server. If that server is hardcoded to port 3000, but your frontend is trying to talk to it via a different port (or if the port is already taken), the fetch will fail.

The Fix:
Check your API server initialization (likely in your spawned process entry point). Ensure it is listening on 0.0.0.0 and not just localhost, as Electron's renderer can sometimes struggle with the local loopback.

TypeScript
// In your server entry point
const PORT = process.env.PORT || 3000;
app.listen(PORT, '0.0.0.0', () => {
  console.log(`API Server running on port ${PORT}`);
});
2. The Vite Proxy Mismatch
If you are running in a development environment, your Vite frontend (port 5173) needs to know how to "forward" requests to your API (port 3000).

The Fix: Update your vite.config.ts:

TypeScript
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:3000',
        changeOrigin: true,
        secure: false,
      }
    }
  }
})
3. PGlite File Locking
You mentioned you want the data self-hosted on the app. PGlite uses a "Single Writer" model.

The Conflict: If your Electron Main process opens the database at %APPDATA%\TCG Snipers\pgdata and then your Spawned API process tries to open the same folder, PGlite will throw a lock error.

The Result: The API process crashes before it can handle the "Save" request, resulting in "Failed to Fetch."

The Fix: Ensure only the spawned API server initializes the PGlite instance. The Main process should only be responsible for spawning the server and passing the path.

4. CORS Middleware
Even if the server is running, the browser (Electron Renderer) might block the request if the headers aren't set.

The Fix: Ensure your server has the cors package installed and active:

TypeScript
import cors from 'cors';
app.use(cors({ origin: '*' }));
How to Confirm the Fix
Open the Electron DevTools (Ctrl+Shift+I).

Go to the Network tab.

Click your "Save" button.

If the status is "Failed": The server isn't reachable (Check the port).

If the status is "CORS Error": The middleware is missing.

If the status is "500 Internal Server Error": PGlite is failing to write to the path (Check for file locks).
